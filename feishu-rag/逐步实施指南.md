# 飞书云文档 RAG 系统 - 逐步实施指南

按顺序完成以下步骤即可完成部署。

---

## 第一步：安装 Python 依赖

**操作：**

1. 打开终端（PowerShell 或 CMD）
2. 进入项目目录：
   ```bash
   cd f:\text2sql\feishu-rag
   ```
3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

**预期结果：** 无报错，显示 `Successfully installed ...`

**若报错：** 可尝试 `pip install openai chromadb python-dotenv -i https://pypi.tuna.tsinghua.edu.cn/simple`

---

## 第二步：创建飞书企业自建应用

**操作：**

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 登录后进入「开发者后台」→「应用开发」→「创建企业自建应用」
3. 填写应用名称（如「知识库问答机器人」）、描述，创建
4. 进入应用详情页，记录：
   - **App ID**（形如 `cli_xxxxxxxxxx`）
   - 点击「凭证与基础信息」→ 生成 **App Secret**，复制保存

**预期结果：** 获得 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET`

---

## 第三步：配置应用权限

**操作：**

1. 在应用详情页，进入「权限管理」
2. 搜索并开通以下权限（勾选后保存）：
   - `docx:document` - 查看、评论、编辑和管理云空间中所有文件
   - `doc:document` - 查看、评论、编辑和管理文档（旧版）
   - `wiki:wiki_node` - 获取知识库节点信息（若使用知识库/Wiki 链接）
   - `bitable:app` - 多维表格权限（若使用多维表格）
   - `im:message` - 获取与发送单聊、群组消息
   - `im:message.group_at_msg` - 接收群聊中 @ 机器人的消息
   - `im:message.p2p_msg` - 接收用户发给机器人的单聊消息

3. 进入「版本管理与发布」→ 创建版本 → 提交审核（或企业内自用可先「仅企业内可用」）

**预期结果：** 权限已开通，应用已发布或企业内可用

---

## 第四步：获取文档 ID

**操作：**

1. 在飞书中打开要作为知识库的云文档或知识库页面
2. 点击右上角「分享」→「复制链接」
3. **支持两种格式：**
   - **普通文档**：`https://xxx.feishu.cn/docx/AbCdEfGhIjKlMnOpQrStUvWx` → 文档 ID 为 `AbCdEfGhIjKlMnOpQrStUvWx`
   - **知识库 Wiki**：`https://xxx.feishu.cn/wiki/WmLKwDNIAiCeHakbXc7cy6bGnHc` → 可直接填**完整链接**，程序会自动提取节点 ID
4. 若有多个文档，用英文逗号拼接

**注意：**
- 知识库需将应用添加为节点协作者或知识空间成员，否则无权限读取
- 普通文档需对应用可见

**预期结果：** 得到 `FEISHU_DOC_IDS` 的值（可填完整链接，程序会自动解析）

---

## 第五步：配置 DeepSeek API Key

**说明：** 本系统参考 wikiqa_rag，使用**本地嵌入模型**（BAAI/bge-small-zh-v1.5），无需 OpenAI。只需配置 **DeepSeek API Key** 用于生成回答。

**操作：**

1. 在 [DeepSeek 开放平台](https://platform.deepseek.com/) 获取 API Key
2. 在 `.env` 中配置：
   ```
   DEEPSEEK_API_KEY=sk-xxxxxxxx
   ```
   （可选）`DEEPSEEK_MODEL=deepseek-chat` 默认已配置

**预期结果：** 仅需 DeepSeek Key，嵌入在本地运行

---

## 第六步：创建 .env 配置文件

**操作：**

1. 在 `feishu-rag` 目录下，复制示例文件：
   ```bash
   copy env_example.txt .env
   ```
   （Linux/Mac 用 `cp env_example.txt .env`）

2. 用记事本或 VS Code 打开 `.env`，按前面步骤填写：
   ```
   FEISHU_APP_ID=cli_你的AppID
   FEISHU_APP_SECRET=你的AppSecret
   FEISHU_DOC_IDS=文档或知识库链接

   DEEPSEEK_API_KEY=sk-xxx
   ```

3. 保存文件

**预期结果：** `.env` 中所有必填项已填写

---

## 第七步：本地测试（不接飞书事件）

**操作：**

1. 在 `feishu-rag` 目录执行：
   ```bash
   python run_bot.py --port 9000 --no-sync-loop
   ```
   `--no-sync-loop` 表示只做一次同步，不启动后台轮询

2. 观察输出：
   - 若出现「正在同步飞书文档...」「文档同步完成」→ 文档拉取成功
   - 若出现「请配置 FEISHU_APP_ID...」→ 检查 `.env`
   - 若出现权限错误 → 检查第三步权限和文档可见范围

3. 在**另一个终端**测试 RAG 问答（可选）：
   ```bash
   cd f:\text2sql\feishu-rag
   python -c "
   from rag_engine import RAGEngine
   r = RAGEngine()
   print(r.query('文档里讲了什么？'))
   "
   ```

**预期结果：** 文档同步成功，RAG 能返回基于文档的回答

---

## 第八步：配置事件订阅（接收飞书消息）

**前提：** 服务需有公网可访问的 URL。本地开发可用 **ngrok** 或 **cpolar** 做内网穿透。

**操作：**

1. **安装 ngrok（示例）**
   - 访问 https://ngrok.com/ 注册并下载
   - 运行：`ngrok http 9000`
   - 记下生成的公网地址，如 `https://abc123.ngrok.io`

2. **在飞书开放平台配置事件**
   - 进入应用 →「事件订阅」
   - Request URL 填写：`https://abc123.ngrok.io/feishu/event`
   - 点击「保存」，飞书会发送校验请求，通过即可

3. **订阅事件**
   - 在事件列表中添加：`im.message.receive_v1`（接收消息）
   - 若需**文档变更实时同步**，再添加：
     - `drive.file.bitable_record_changed_v1`（多维表格记录变更）
     - `drive.file.edit_v1`（文档编辑，可选）
   - 保存配置

**预期结果：** 事件订阅配置成功，校验通过

**注意：** 云文档事件订阅需应用为文档**拥有者或管理员**。启动后会自动调用订阅 API，多维表格/文档有变更时飞书会推送，RAG 将实时同步。

---

## 第九步：将机器人加入群聊并测试

**操作：**

1. 在飞书中创建一个群（或使用已有群）
2. 群设置 → 群机器人 → 添加机器人 → 选择你创建的企业自建应用
3. 在群里 @机器人 并提问，例如：
   ```
   @知识库问答机器人 文档里有哪些主要内容？
   ```

4. 若一切正常，机器人会基于文档内容回复

**预期结果：** 机器人能正确接收问题并返回 RAG 回答

---

## 第十步：正式部署（可选）

**操作：**

1. 将服务部署到有公网 IP 的服务器（如云主机）
2. 使用 `python run_bot.py --port 9000` 启动（不加 `--no-sync-loop`）
3. 将飞书事件订阅 URL 改为服务器的公网地址
4. 建议用 systemd 或 supervisor 做进程守护，保证服务常驻

**预期结果：** 服务稳定运行，文档定期同步，机器人可持续应答

## 常见问题

| 问题 | 可能原因 | 处理方式 |
|------|----------|----------|
| 文档同步失败 | 无权限 / 文档 ID 错误 | 检查权限、文档链接和 ID |
| 机器人无回复 | 事件未收到 / 未 @ 机器人 | 见下方「机器人无回复排查」 |
| 嵌入报错 | 本地模型未下载 | 首次运行会下载 BGE 模型，需网络 |
| 校验失败 | URL 不可达 / 端口错误 | 确认 ngrok 或公网地址可访问 |

### 机器人无回复排查

1. **开启调试**：启动前设置 `set FEISHU_DEBUG=1`（Windows）或 `export FEISHU_DEBUG=1`（Linux），再运行 `python run_bot.py`
2. **确认 @ 机器人**：群聊中必须 @ 机器人 才会触发，直接发文字不会
3. **看 ngrok 请求**：访问 http://127.0.0.1:4040 查看是否有 POST 请求到达
4. **看终端输出**：若有 `[DEBUG] received event` 说明收到事件；若有 `[DEBUG] skip` 说明事件类型不匹配

## 步骤总览

```
1. 安装依赖 (pip install)
2. 创建飞书应用 → 获取 App ID / Secret
3. 配置应用权限
4. 获取文档 ID
5. 配置 DeepSeek API Key
6. 创建 .env
7. 本地测试（run_bot.py --no-sync-loop）
8. 配置事件订阅（ngrok + 飞书 Request URL）
9. 加群测试 @机器人
10. 正式部署（可选）
```
